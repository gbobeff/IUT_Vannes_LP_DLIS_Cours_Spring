Introduction

  • développé par Rod Johnson (2003)
    • 2002 publia son livre « Expert One-on-One J2EE Design and Development »
    • Rod n’aimait pas la complexité et la difficulté à maintenir de EJB 2.0 et décida de développer un framework plus simple pour les applications web en J2EE.
    • dans son livre, il présentait les bases et les principes de ce framework J2EE alternatif, et fournissait 30000 lignes de code source pour l’accompagner.
  • lightweight framework (cadriciel léger)
    • Framework des frameworks (Struts, Hibernate, JSF, ...)
  • une véritable plateforme pour développer
  • et libre !!! (License Apache 2.0)

Chronologie

  • juin 2003, la première version est apparue sous le nom version 0.9
  • en mars 2004 sort la version officielle 1.0 de Spring
    • le but? à l’époque était de rendre les architectures d’entreprise complexes plus faciles à tester et à maintenir
    • se base sur les conventions JavaBeans pour ses modèles de composants
    • depuis le début, Spring pouvait être utilisé sur tous les serveurs applicatifs ou conteneurs web classiques aussi bien que dans un environnement J2SE (c’est-à-dire sans conteneurs J2EE)
  • en septembre 2004, la version 1.1 est sortie avec des bugs corrigés et de nouvelles fonctions.
  • la version 1.2, sortie en mai 2005, supportait déjà quelques fonctions de Java 5 tout en maintenant une rétrocompatibilité avec les versions de Java plus anciennes
  • en octobre 2006 sort la version 2.0 qui allait apporter beaucoup d’innovation
    • à cette époque, le framework avait été téléchargé un million de fois et gagné deux récompenses : un Jax Innovation Award et un Jolt en 2006
  • à ce jour, Spring a connu plusieurs développements et plusieurs ajouts, les plus importants ci-dessous :
    • 2007 — Spring Framework 2.5 introduit les configurations avec annotation
    • 2009 — Spring Framework 3.0 introduit les classes de configuration Java et requiert a minima Java 5
      • achat de SpringSource par VMWare
    • 2013 — Spring Framework 4.0 supporte complètement Java 8
    • 2017 — Spring Framework 5.0 supporte Kotlin et les architectures réactives, et requiert a minima Java 8.

Le projet Spring Boot

  • en octobre 2012, suite à la demande d'un client une extension a été introduite au Spring Framework
    • demande de fonction qui nécessitait une architecture d'application Web sans conteneur
  • cela conduisit au développement du projet Spring Boot au début de 2013 et finalement à la sortie de Spring Boot 1.0 en avril 2014.
  • les problèmes qu’il traite sont :
    • les applications Spring et Java comprennent un grand nombre de librairies (de tiers) techniques pour résoudre des problèmes comme la sécurité
    • la sélection des librairies et, plus particulièrement, le choix des versions compatibles de celles-ci était à la charge des équipes de développement
  • solution :
    • fourniture d'un modèle BOM (Bill of Material) pour des projets uniques
    • les équipes de développement accèdent à une liste mise à jour de dépendances en se basant sur les aspects techniques (web, batch, base de données) dans les versions compatibles et les mises à jour Spring Bot et puis poussent les mises à jour des librairies tierces
